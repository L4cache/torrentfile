    Using

    ```bash: usage: torrentfile -v --version torrentfile -h --help torrentfile --checker torrentfile [-o ] [-a ] [-d] [--source ] [--piece-length ] [--meta-version ] [--private] [--announce-list <...>] [--comment ]

    Create Bittorrent meta files for Bittorrent v1 and v2.

    positional arguments: Path to content source file or directory.

    optional arguments: -h, --help show this help message and exit -v, --version Show program version and exit -d, --debug Turn on debug mode -a , --announce Primary announce url for Bittorrent Tracker. -p , --piece-length 

                                        Intiger piece length for content used by Bittorrent Protocol.
                                    Acceptable Input values include 14-35 which will be treated as
                                    an exponent for 2^n power. Otherwise the value must be a
                                    perfect power of 2 between 16KB and 16MB.
                                    i.e. [--piece-length 14] is the same as [--piece-length  16384]
                                    Alternatively, leave blank and let the program calulate the
                                    appropriate piece length.

    --private Create file for use with private tracker. -o , --out Path to the destination for the output .torrent file. --meta-version Options = 1, 2 or 3. (1) = Bittorrent v1;. (Default) (2) = Bittorrent v2. (3) = Bittorrent v1 & v2 hybrid. Specify the Bittorrent Protocol and formatting version for .torrent file.

    --comment Include a comment in file metadata. --source Specify source. --announce-list [ ...] Additional tracker announce URLs. --re-check 

                                        Trigger torrent re-check mode.
                                    When option is active, all other options are ignored (except debug).
                                    This option forces a re-check on specified torrent contents to
                                    calculate an accurate percentage complete for downloaded content.
                                    Arguements:
                                    1) Absolute or relative path to The path to a ".torrent" file.
                                    2) Absolute or relative path to Torrent Contents

    ```

    CLI Usage Examples

    Example 1

    ```bash
$ torrentfile /path/to/data --announce https://tracker.com/announce_1234 --private
```
    • Create a .torrent file with:
      • contents: /path/to/data
      • tracker : https://tracker.com/announce_1234
      • set for private tracker.

    Example 2

    ```bash
$ torrentfile /contnts/path -d --announce-list https://traker2.com/annce https://traker3.com/annce --piece-length 18
```
    • Create .torrent file in debug mode with:
      • contents: /some/dir
      • trackers: https://tracker1.com/announce, https://tracker2.com/announce, https://tracker3.com/announce
      • piece length: 262144 or 218 bytes

    Example 3

    ```bash
$ torrentfile /home/document -o /home/document_1.torrent -a https://tracker.com --meta-version 3
```
    • Create a .torrent file with Bittorrent v1 & 2 Hybrid format:
      • contents: /home/document
      • tracker: https://tracker.com
      • output file: /home/document_1.torrent

    Example 4

    ```bash
$ torrentfile -d --checker /path/to/file.torrent /path/to/contents
>> 100%
```
    • Check a .torrent file in debug mode against directory contents.
